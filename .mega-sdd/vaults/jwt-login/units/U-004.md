# Unit U-004 — Create DTOs (LoginRequest, JwtResponse, MessageResponse)

## Goal

Create the three API-boundary DTOs replicating newmojf's response/request shapes: `LoginRequest{uname,pass}`, `JwtResponse{token,type,id,uname,mitKode,urole}`, `MessageResponse{message}`.

## Context

- **Vault source:** `02-architecture.md` §Backend (DTO component) + §API contracts (response shape); `04-flows.md` F-U-001.
- **Binding refs:** C-008, OQ-AR-7 (DTO field naming — verbatim newmojf `mitKode`/`urole`/`type` vs rename to `kodeMitra`/`role`; recommendation PENDING — default to verbatim newmojf replication, surface if user overrides).
- **Why this scope:** DTOs decouple API shape from entity schema (pack Hard Rule: no entity in REST response). Field names VERBATIM from `JwtResponse.java:5-10` (token, type="Bearer ", id, uname, mitKode, urole) — the advisor ADV-001 fabrication fix locked this.

## Implementation steps

1. Create `dto/LoginRequest.java`: fields `String uname`, `String pass` + getters/setters (no Lombok).
2. Create `dto/JwtResponse.java` replicating `JwtResponse.java:3-19` EXACTLY: fields `String token`, `String type = "Bearer "`, `Long id`, `String uname`, `String mitKode`, `String urole`; constructor `(String token, Long id, String uname, String mitKode, String urole)`; getters/setters. NO `@JsonProperty` (newmojf has none — Jackson serializes bare field names).
3. Create `dto/MessageResponse.java`: field `String message` + constructor + getter/setter (replicates newmojf `MessageResponse`).
4. `./mvnw compile` to confirm.

## Acceptance criteria

- All three DTOs compile; located in `dto/` package.
- `JwtResponse` fields are EXACTLY `{token, type="Bearer ", id, uname, mitKode, urole}` (verbatim newmojf — not `kodeMitra`/`role`).
- `JwtResponse` constructor signature matches `JwtResponse.java:12`.
- No `@JsonProperty` annotations (matches newmojf; bare-field Jackson serialization).

- DO NOT rename `mitKode`→`kodeMitra` or `urole`→`role` unless OQ-AR-7 resolves to rename (advisor ADV-001: verbatim replication is the default; renaming is an explicit adaptation decision).

- DO NOT add `@JsonProperty` unless an OQ decides on a different JSON contract.
