# Unit U-002 — Create Users entity (jakarta.persistence)

## Goal

Create the `Users` JPA entity replicating the `mojf_users_Model` field set (18 fields), adapted to `jakarta.persistence.*` (Boot 4.x) and the pack `entity/` package location.

## Context

- **Vault source:** `03-data-model.md` §Entities (DBML, 18 fields); `02-architecture.md` §Backend component (Users entity).
- **Binding refs:** C-003 (entity users 18 fields jakarta.persistence), C-004 (entity/ package), OQ-DM-1 (table name `users` vs `mojf_users` — recommendation: `users` unless OQ-AR-2 picks newmojf DB), OQ-DM-3 (ddl-auto=none — recommendation ACCEPTED).
- **Why this scope:** Entity is the lookup target for login (`findByUname`). Replicates newmojf `mojf_users_Model.java` field-by-field but with `jakarta.persistence` namespace (the #1 migration hazard — `javax` won't compile on Boot 4.x).

## Implementation steps

1. Create `src/main/java/com/coresystem/coresystembackend/entity/Users.java`.
2. Annotate `@Entity`, `@Table(name = "users")` (per OQ-DM-1 recommendation — `users` pack standard; if OQ-AR-2 resolves to newmojf DB, change to `mojf_users` before bolt runs).
3. Field `id` (Long): `@Id @GeneratedValue(strategy = GenerationType.IDENTITY) @Column(name = "uid_")`.
4. Fields with explicit `@Column(name = ...)` snake_case mapping (camelCase Java → snake_case DB): `uname`, `pass`, `namaLengkap`→`nama_lengkap`, `kodeUnitKerja`→`kode_unit_kerja`, `cakupan` (Long), `urole` (Long), `createdDate`→`created_date` (java.util.Date), `createdBy`→`created_by`, `lastModified`→`last_modified`, `modifiedBy`→`modified_by`, `lastLogin`→`last_login` (Date), `kodeMitra`→`kode_mitra`, `active` (Long), `expiredDays`→`expired_days` (Long), `kodeKelasUser`→`kode_kelas_user`, `statusUser`→`status_user` (Long).
5. Use `jakarta.persistence.*` imports — NOT `javax.persistence.*`.
6. Generate getters/setters (no Lombok — OQ-CN-2 deferred). Constructor injection N/A (entity is a POJO).
7. `./mvnw compile` to confirm.

## Acceptance criteria

- `Users.java` compiles under Boot 4.x with `jakarta.persistence` imports.
- All 18 fields present with correct `@Column(name=...)` snake_case mapping matching `mojf_users_Model.java:15-50`.
- `@Entity @Table(name="users") @Id @GeneratedValue(IDENTITY)` present.
- No `javax.persistence` import anywhere in the file.

## Anti-patterns

- DO NOT use `javax.persistence.*` (constitution §B-001) — must be `jakarta.persistence.*`.
- DO NOT add business logic / JPA callbacks to the entity (pack: entity is a POJO; constitution §C-003 — logic in @Service).
- DO NOT update `last_login` on login (OQ-DM-2 — v1 read-only replication; newmojf `/dologin` doesn't write it).
- Table name `users` is the default; flip to `mojf_users` ONLY if OQ-AR-2 resolves to the newmojf existing DB (OQ-DM-1 dependency).
