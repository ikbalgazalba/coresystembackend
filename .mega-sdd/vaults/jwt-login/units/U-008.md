# Unit U-008 — Create AuthUserController + application.yaml config

## Goal

Create `AuthUserController` (`POST /api/auth/dologin`) replicating newmojf's `/dologin` flow: LDAP auth → JWT from uname → user lookup → `JwtResponse`. Also configure `application.yaml` with JWT secret/expiration + PostgreSQL datasource.

## Context

- **Vault source:** `02-architecture.md` §API contracts (`POST /api/auth/dologin`); `04-flows.md` F-U-001; `06-constraints.md` §Technical (config).
- **Binding refs:** C-001 (controller), C-002 (endpoint), C-011 (config), OQ-AR-2 (target DB — P1 blocking, affects datasource URL), OQ-AR-3 (jwtSecret env var — ACCEPTED), OQ-AR-7 (DTO field naming — verbatim mitKode/urole default), OQ-FL-3 (error body sanitization).
- **Why this scope:** This is the integration point wiring U-003/004/005/007 into the login endpoint. It also owns the config (datasource + JWT keys).

## ⚠️ P1 BLOCKER — execute-bolts will HALT this unit unless OQ-AR-2 is resolved

The PostgreSQL datasource URL/credentials depend on OQ-AR-2 (newmojf existing DB vs new coresystembackend DB). The bolt can write the controller + JWT config, but the datasource block needs the OQ-AR-2 decision before the app boots against a real DB. Implement with a `${...}` placeholder datasource and flag the halt.

## Implementation steps

1. Create `src/main/java/com/coresystem/coresystembackend/controller/AuthUserController.java`, `@RestController @RequestMapping("/api/auth")`.
2. Constructor-inject `LdapUcsService`, `JwtUtils`, `UserRepository` (NO field `@Autowired` — constitution §C-004/D-005).
3. `@PostMapping("/dologin") authenticateUserUCS(@RequestBody LoginRequest req)`:
   - `LdapAuthResult r = ldapUcsService.authLDAPNew(req.getUname(), req.getPass())`.
   - If `r == null` → `400 MessageResponse("Failed to connect to LDAP service")`.
   - If `responseCode` is "00" or "01": `String jwt = jwtUtils.generateTokenFromUname(req.getUname())`; `Users usr = userRepository.findByUname(req.getUname()).orElseThrow(...)`; `String urole = "ROLE_" + usr.getUrole()`; return `200 new JwtResponse(jwt, usr.getId(), usr.getUname(), usr.getKodeMitra(), urole)`.
   - Else → `400 MessageResponse(<sanitized description>)` (OQ-FL-3 — do NOT echo raw upstream `responseDescription` if it may contain internal error text; map to generic unless OQ-FL-3 resolves to verbatim).
   - Wrap in try/catch → `401 MessageResponse("Authentication failed")` on exception (matches newmojf `AuthUserController.java:140-142`).
4. Do NOT add `@CrossOrigin` (CORS handled by U-006 SecurityConfig).
5. Modify `application.yaml`: add `coresystem.app.jwtSecret: ${JWT_SECRET:dev-only-secret-change-me}` (env-var placeholder — OQ-AR-3), `coresystem.app.jwtExpirationMs: 86400000`, `spring.datasource.url/username/password` (PostgreSQL `${...}` placeholders pending OQ-AR-2), `spring.jpa.hibernate.ddl-auto: none` (OQ-DM-3), `spring.jpa.show-sql: false`. Keep existing `spring.application.name`.
6. Write `AuthUserControllerTest` (`@WebMvcTest(AuthUserController.class)` with `@MockBean` LdapUcsService/JwtUtils/UserRepository): success (200 + JwtResponse fields `token,type,id,uname,mitKode,urole`), bad-cred (400), null-ldap (400 "Failed to connect"), exception (401).
7. `./mvnw test -Dtest=AuthUserControllerTest`.

## Acceptance criteria

- `POST /api/auth/dologin` returns 200 `JwtResponse{token, type="Bearer ", id, uname, mitKode, urole}` for valid creds (field names VERBATIM — advisor ADV-001).
- Bad credentials → 400 `MessageResponse`; LDAP null → 400 "Failed to connect to LDAP service"; exception → 401 "Authentication failed".
- `application.yaml` has `coresystem.app.jwtSecret` (env-var placeholder, NOT literal), `jwtExpirationMs: 86400000`, PostgreSQL datasource placeholders, `ddl-auto: none`.
- Constructor injection only; no `@CrossOrigin`; no field `@Autowired`.
- `AuthUserControllerTest` passes.

## Anti-patterns

- DO NOT return the `Users` entity from the controller (constitution §C-002/D-006 — use JwtResponse DTO).
- DO NOT put LDAP/JWT/lookup orchestration logic inline as business logic — delegate to `LdapUcsService`/`JwtUtils`/`UserRepository`; controller only parses + delegates (constitution §C-003).
- DO NOT use field `@Autowired` (constitution §C-004/D-005 — constructor injection).
- DO NOT hardcode `jwtSecret` literal (constitution §D-002, OQ-AR-3).
- DO NOT echo raw LDAP `responseDescription` if it may leak internals (constitution §B-007, OQ-FL-3).
- DO NOT rename JwtResponse fields to `kodeMitra`/`role` unless OQ-AR-7 resolves to rename (advisor ADV-001 — verbatim default).
