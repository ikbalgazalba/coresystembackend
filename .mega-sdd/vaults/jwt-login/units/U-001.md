# Unit U-001 — Add Maven dependencies for web, security, JPA, PostgreSQL, JWT

## Goal

Extend the skeleton `pom.xml` (currently only `spring-boot-starter` + `-test`) with the starters and libraries the JWT-login feature requires: web, security, data-jpa, PostgreSQL driver, and jjwt.

## Context

- **Vault source:** `06-constraints.md` §Technical constraints (dependencies to add); `02-architecture.md` §starterkit binding.
- **Binding refs:** C-010 (dependencies to add), OQ-AR-4 (jjwt version + legacy API rewrite — recommendation ACCEPTED: jjwt 0.12.x triplet).
- **Why this scope:** No HTTP/security/persistence layer exists yet (`codebase-map.md §7`: only `spring-boot-starter`). This unit is the foundation every other unit compiles against. It touches only `pom.xml` — no Java code.

## Implementation steps

1. Add `spring-boot-starter-web` dependency (compile scope).
2. Add `spring-boot-starter-security` dependency (compile scope).
3. Add `spring-boot-starter-data-jpa` dependency (compile scope).
4. Add `org.postgresql:postgresql` dependency (runtime scope).
5. Add the jjwt triplet: `io.jsonwebtoken:jjwt-api` (compile) + `io.jsonwebtoken:jjwt-impl` (runtime) + `io.jsonwebtoken:jjwt-jackson` (runtime), version `0.12.x` (resolve exact patch via `./mvnw dependency:tree` against the snapshot repo — pick the latest 0.12.x that resolves).
6. (Optional, deferred per OQ-CN-2) Do NOT add Lombok unless OQ-CN-2 resolves to ACCEPT — use SLF4J manual `LoggerFactory` in the meantime.
7. Keep the existing `spring-snapshots` repository block intact (Boot 4.1.1-SNAPSHOT resolution requires it).
8. Run `./mvnw -q dependency:resolve` then `./mvnw -q compile` to confirm the classpath resolves over the snapshot repo.

## Acceptance criteria

- `./mvnw dependency:resolve` exits 0 (all deps resolve from repo.spring.io/snapshot + Maven Central).
- `./mvnw compile` exits 0 (existing `CoresystembackendApplication` still compiles with new deps on classpath).
- `pom.xml` declares: spring-boot-starter-web, spring-boot-starter-security, spring-boot-starter-data-jpa, postgresql (runtime), jjwt-api + jjwt-impl + jjwt-jackson 0.12.x.
- No Lombok added (deferred to OQ-CN-2).

## Hard rules

- DO NOT add new package.json dependencies
  Citation: constitution §D-003 (DO_NOT_ADD_DEPS)

> (Above is the only parseable pack Hard Rule for this unit — `pom.xml` has no package.json; this is a no-op guard. Real dependency-review discipline is enforced by constitution D-003 anti-pattern below.)

## Anti-patterns

- DO NOT add dependencies beyond the feature's declared set (constitution §D-003: no new dep without review). Lombok is explicitly deferred (OQ-CN-2).
- DO NOT pin a non-0.12.x jjwt version — newmojf's legacy `signWith(SignatureAlgorithm, String)` API is removed in 0.11+; U-005 requires the 0.12.x API (`Jwts.builder().signWith(Key)`, `Jwts.parserBuilder().verifyWith(key)`).

## Out of scope

- Java source code (entity, controller, security config) — later units.
- `application.yaml` datasource/JWT config — U-008.
- Exact jjwt patch version pinning if 0.12.x latest doesn't resolve — flag to user.
