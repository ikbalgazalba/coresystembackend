# Unit U-005 — Create JwtUtils (jjwt 0.12.x API)

## Goal

Create `JwtUtils` replicating newmojf's `generateJwtTokenUname(uname)` + validate/parse, but REWRITTEN to the jjwt 0.12.x API (newmojf uses the legacy `signWith(SignatureAlgorithm, String)` removed in 0.11+).

## Context

- **Vault source:** `02-architecture.md` §Backend (JwtUtils); `04-flows.md` F-S-001 (generation + validation wiring).
- **Binding refs:** C-006, OQ-AR-3 (jwtSecret externalize via env var — recommendation ACCEPTED), OQ-AR-4 (jjwt 0.12.x + legacy API rewrite — VALIDATED recommendation ACCEPTED).
- **Why this scope:** newmojf `JwtUtils.java:42-50` uses `Jwts.builder().signWith(SignatureAlgorithm.HS512, jwtSecret)` and `:53,58` `Jwts.parser().setSigningKey(jwtSecret)` — both removed in jjwt 0.11+. This unit replicates the BEHAVIOR (HS512-signed JWT, subject=uname, exp from config) with the modern API.

## Implementation steps

1. Create `src/main/java/com/coresystem/coresystembackend/security/JwtUtils.java`, `@Component`.
2. Inject `@Value("${coresystem.app.jwtSecret}") String jwtSecret` and `@Value("${coresystem.app.jwtExpirationMs}") int jwtExpirationMs` (config keys defined in U-008 application.yaml; secret via env-var placeholder per OQ-AR-3).
3. Build the signing key: `Key key = Keys.hmacShaKeyFor(Decoders.BASE64.decode(jwtSecret))` — NOTE: jjwt 0.12.x requires a `Key`, not a raw String. If the secret is too short for HS512, use `Keys.hmacShaKeyFor(jwtSecret.getBytes())` with HS256 (flag in acceptance — secret length matters).
4. `generateTokenFromUname(String uname)`: `Jwts.builder().header().add("typ","JWT").and().subject(uname).issuedAt(new Date()).expiration(new Date(now + jwtExpirationMs)).signWith(key, Jwts.SIG.HS512).compact()`.
5. `getUserNameFromJwt(String token)`: `Jwts.parser().verifyWith(key).build().parseSignedClaims(token).getPayload().getSubject()`.
6. `validateJwtToken(String authToken)`: try parse, return false on any `JwtException`/`IllegalArgumentException`; log via SLF4J `LoggerFactory` (no `System.out`, no Lombok).
7. Write `JwtUtilsTest` (unit test): generate token from "testuser", parse subject back, validate returns true, tampered token validates false.
8. `./mvnw test -Dtest=JwtUtilsTest`.

## Acceptance criteria

- `JwtUtils` compiles against jjwt 0.12.x (no `SignatureAlgorithm`, no `signWith(SignatureAlgorithm,*)`, no `Jwts.parser().setSigningKey`).
- `generateTokenFromUname("testuser")` produces a non-empty signed JWT; `getUserNameFromJwt(token)` returns "testuser".
- `validateJwtToken(tamperedToken)` returns false (no exception thrown to caller).
- `jwtSecret` read from config (not hardcoded literal — OQ-AR-3).
- `JwtUtilsTest` passes.

## Anti-patterns

- DO NOT use `SignatureAlgorithm.HS512` or `signWith(SignatureAlgorithm, String)` (removed in jjwt 0.11+ — OQ-AR-4 legacy API hazard).
- DO NOT use `Jwts.parser().setSigningKey(String)` — use `Jwts.parser().verifyWith(Key).build()` (0.12.x).
- DO NOT hardcode `jwtSecret` literal (constitution §D-002, OQ-AR-3 — externalize via `${JWT_SECRET:...}` placeholder in U-008).
- DO NOT use `System.out.println` for logging (pack forbidden — SLF4J `LoggerFactory`).
