# Unit U-009 — Integration test (login flow end-to-end)

## Goal

Write a `@SpringBootTest` + `MockMvc` integration test covering F-U-001 (login → JWT issuance) end-to-end, asserting the DoD from `04-flows.md`.

## Context

- **Vault source:** `04-flows.md` F-U-001 (Login JWT issuance) — Definition of Done (9 items).
- **Binding refs:** C-002 (endpoint), F-U-001.
- **Why this scope:** The DoD is the QA contract; this unit encodes it as an executable test. Validates the wiring of U-003/004/005/007/008 together with mocked LDAP (no real LDAP UCS — OQ-AR-1 unresolved).

## Implementation steps

1. Create `src/test/java/com/coresystem/coresystembackend/AuthLoginIntegrationTest.java`, `@SpringBootTest @AutoConfigureMockMvc`.
2. `@MockBean LdapUcsService`, `@MockBean UserRepository` (isolate from real LDAP/DB — OQ-AR-1/OQ-AR-2 unresolved).
3. Test `loginSuccess_returnsJwtResponse` : mock `ldapUcsService.authLDAPNew` → `LdapAuthResult("00","OK")`; mock `userRepository.findByUname` → `Users(id=1, uname="alice", kodeMitra="M01", urole=2L)`; `mockMvc.perform(post("/api/auth/dologin").contentType(APPLICATION_JSON).content("""{"uname":"alice","pass":"x"}"""))` → `status().isOk()` + jsonPath `$.token` exists, `$.type` = "Bearer ", `$.id` = 1, `$.uname` = "alice", `$.mitKode` = "M01", `$.urole` = "ROLE_2".
4. Test `loginBadCredentials_returns400` : mock responseCode "99" → `status().isBadRequest()` + `$.message` present.
5. Test `loginLdapNull_returns400FailedToConnect` : mock `authLDAPNew` → null → `isBadRequest()` + `$.message` = "Failed to connect to LDAP service".
6. Test `loginException_returns401` : mock `authLDAPNew` throws → `isUnauthorized()` + `$.message` = "Authentication failed"; assert response body does NOT contain raw exception text (constitution §B-007).
7. `./mvnw test -Dtest=AuthLoginIntegrationTest`.

## Acceptance criteria

- All 4 test cases pass.
- Success response asserts the VERBATIM JwtResponse field names (`token, type, id, uname, mitKode, urole`) — NOT `kodeMitra`/`role` (advisor ADV-001 regression guard).
- Exception-path response body contains no raw exception text (constitution §B-007 / OQ-FL-3 guard).
- LDAP + DB are mocked (no real external dependency — OQ-AR-1/AR-2 unresolved).
