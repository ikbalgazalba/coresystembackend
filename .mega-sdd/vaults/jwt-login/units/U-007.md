# Unit U-007 — Create LdapUcsService (LDAP UCS replication)

## Goal

Create `LdapUcsService` replicating newmojf `LDAP_UCS_Utils.authLDAPNew(uname, pass)` → `{responseCode, responseDescription}` (success codes "00"/"01"), adapted to constructor injection and WITHOUT replicating the `disableSslVerification()` trust-all anti-pattern.

## Context

- **Vault source:** `02-architecture.md` §Backend (LdapUcsService) + §Integrations (LDAP UCS outbound); `05-decisions.md` D-001.
- **Binding refs:** C-009, OQ-AR-1 (LDAP infra aktif — P1 blocking), OQ-AR-6 (prasyarat: AES key, client cred, URLs — P1 blocking), OQ-FL-1 (kontrak response LDAP UCS — P1 blocking), OQ-FL-3 (sanitasi error body — info-leakage).
- **Why this scope:** The reference impl (`LDAP_UCS_Utils.java`) has substantial hidden behavior: AES/ECB password encryption (`:267-279`), HMAC-SHA512 signature (`:286-319`), `DB_Connection`-sourced credentials/URLs, and a `disableSslVerification()` trust-all (`:42-61`). Replicating the auth contract requires the AES key + client credentials + token/verify URLs to be provisioned (OQ-AR-6) — **without these, the service cannot function at runtime**.

## ⚠️ P1 BLOCKER — execute-bolts will HALT this unit unless OQ-AR-1, OQ-AR-6, OQ-FL-1 are resolved

This unit implements against an external LDAP UCS endpoint whose existence (OQ-AR-1), prerequisites (OQ-AR-6), and response contract (OQ-FL-1) are UNRESOLVED P1 business OQs. The bolt SHOULD implement the service skeleton + a mock/stub path for dev, but MUST surface the halt if the P1s are unresolved when it runs. The implementer does not guess endpoint URLs or credentials.

## Implementation steps

1. Create `src/main/java/com/coresystem/coresystembackend/service/LdapUcsService.java`, `@Service`.
2. Constructor-inject config values: `urlToken`, `urlVerifyPassword`, `clientId`, `clientSecret`, `aesKey`, `partnerId`, `channelId`, `host` — all via `@Value("${coresystem.ldap.*}")` placeholders (undefined until OQ-AR-6 provisions them; defaults empty / profile-gated).
3. Define a result record/class `LdapAuthResult(String responseCode, String responseDescription)`.
4. `LdapAuthResult authLDAPNew(String uname, String pass)`:
   - AES/ECB/PKCS5Padding-encrypt the password (replicate `encryptPassword`, `:267-279`) using the configured `aesKey`.
   - Build the HMAC-SHA512 signature (replicate `:286-319`).
   - Call `getToken(...)` then `getLdapProcess(...)` via an injected `RestClient`/`RestTemplate` (Spring 7.x — prefer `RestClient`).
   - Parse the upstream response into `{responseCode, responseDescription}`; success = "00" or "01".
   - On internal exception: return `LdapAuthResult("401", <generic message>)` — **DO NOT echo raw `e.getMessage()`** (constitution §B-007, OQ-FL-3, advisor ADV-005 — information-leakage). Log the raw detail server-side via SLF4J.
5. **DO NOT** replicate `disableSslVerification()` (constitution §B-006, advisor ADV-006). If the endpoint requires custom SSL, raise an OQ — do not install a trust-all `X509TrustManager`/`HostnameVerifier`.
6. Write `LdapUcsServiceTest` with a MOCKED HTTP layer (MockRestServiceServer / MockWebServer) — assert success path (responseCode "00" → returns result), bad-cred path (responseCode "99" → returns result with description), exception path (returns generic "401", no raw message leaked).
7. `./mvnw test -Dtest=LdapUcsServiceTest`.

## Acceptance criteria

- `LdapUcsService` compiles; constructor injection (no field `@Autowired`).
- `authLDAPNew` returns `{responseCode, responseDescription}`; success codes "00"/"01" honored.
- Exception path returns a GENERIC message (not `e.getMessage()`) — `LdapUcsServiceTest` asserts the response body does NOT contain raw exception text.
- NO `disableSslVerification` / trust-all / empty `X509TrustManager.checkServerTrusted` anywhere in the codebase.
- All external config (URLs/creds/keys) read via `@Value` placeholders — none hardcoded.
- `LdapUcsServiceTest` passes.

## Anti-patterns

- DO NOT replicate `disableSslVerification()` trust-all (constitution §B-006, advisor ADV-006) — security anti-pattern grade constitution.
- DO NOT echo `responseDescription`/`e.getMessage()` verbatim to the client (constitution §B-007, OQ-FL-3, advisor ADV-005) — map to generic, log raw server-side.
- DO NOT hardcode AES key / client credentials / URLs (constitution §D-002 — externalize via config; pending OQ-AR-6 provisioning).
- DO NOT put this service's logic in the controller (constitution §C-003 — business logic in @Service).
