# Unit U-006 — Create SecurityConfig (Spring Security 7.x)

## Goal

Create `SecurityConfig` with a `@Bean SecurityFilterChain` (NOT `WebSecurityConfigurerAdapter`) — permitAll `/api/auth/**`, stateless sessions, CSRF disabled for the stateless token API, a `PasswordEncoder` bean, and CORS via `SecurityFilterChain` (not wildcard `@CrossOrigin`).

## Context

- **Vault source:** `02-architecture.md` §Backend (SecurityConfig); `05-decisions.md` D-004 (verified: newmojf `security_Config.java:54 extends WebSecurityConfigurerAdapter` — we do NOT replicate that).
- **Binding refs:** C-007, OQ-AR-5 (CORS — recommendation ACCEPTED: SecurityFilterChain `http.cors` + `CorsConfigurationSource` with explicit origins, not wildcard).
- **Why this scope:** Spring Security 7.x removed `WebSecurityConfigurerAdapter`; the pack forbids it. This unit establishes the security posture every endpoint inherits.

## Implementation steps

1. Create `src/main/java/com/coresystem/coresystembackend/config/SecurityConfig.java`, `@Configuration @EnableWebSecurity`.
2. `@Bean SecurityFilterChain filterChain(HttpSecurity http)`: `http.csrf(csrf -> csrf.disable())` WITH an explanatory comment ("stateless JWT API — CSRF not applicable"), `.sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))`, `.authorizeHttpRequests(a -> a.requestMatchers("/api/auth/**").permitAll().anyRequest().authenticated())`, `.cors(Customizer.withDefaults())`.
3. `@Bean PasswordEncoder passwordEncoder()`: return `PasswordEncoderFactories.createDelegatingPasswordEncoder()` (constitution §B-005 — bean present even though auth is via LDAP).
4. `@Bean CorsConfigurationSource corsConfigurationSource()`: `CorsConfiguration` with `setAllowedOrigins(List.of(...))` (explicit list — NOT `*`; origins TBD pending OQ-AR-5 finalization, default to a dev origin), `setAllowedMethods(List.of("POST","GET","PUT","DELETE","OPTIONS"))`, `setAllowedHeaders(List.of("*"))`, `applyPermitDefaultValues()` optional; register under `UrlBasedCorsConfigurationSource` for `/**`.
5. `@Bean AuthenticationManager authenticationManager(AuthenticationConfiguration cfg)`: expose if needed (newmojf's `/dologinOld` used it; `/dologin` active path does not — keep the bean optional/unused, or omit). Prefer OMIT unless U-008 needs it.
6. Write `SecurityConfigTest`: `@WebMvcTest` or `@SpringBootTest` — assert `/api/auth/dologin` is reachable without auth (permitAll), and a protected path (e.g. `/api/anything-else`) returns 401/403.
7. `./mvnw test -Dtest=SecurityConfigTest`.

## Acceptance criteria

- `SecurityConfig` compiles; uses `SecurityFilterChain` bean (NO `extends WebSecurityConfigurerAdapter`).
- `/api/auth/**` is permitAll; everything else `authenticated()`.
- Session policy STATELESS; CSRF disabled with explanatory comment.
- `PasswordEncoder` bean present (delegating).
- `CorsConfigurationSource` bean present with EXPLICIT allowed origins (no `*`).
- `SecurityConfigTest` passes.

## Hard rules

- file src/main/java/com/coresystem/coresystembackend/config/SecurityConfig.java MUST exist after bolt
  Citation: binding.md §Suggested Unit Hard Rules (FILE_PRESENCE_RULE)

## Anti-patterns

- DO NOT `extends WebSecurityConfigurerAdapter` (constitution §B-002, pack forbidden — removed in Spring 6).
- DO NOT use `@CrossOrigin(origins = "*")` (constitution §C via OQ-AR-5, advisor ADV-004) — CORS via SecurityConfig.
- DO NOT use `NoOpPasswordEncoder` (pack security idiom — use delegating BCrypt).
- DO NOT disable CSRF without an explanatory comment (pack security idiom §CSRF).

## Out of scope

- JWT validation filter for protected endpoints (future — D-002 keeps token uname-only; resource-server is out of v1 scope per `02-architecture.md` Out of Scope).
- Method-level security `@EnableMethodSecurity` (no protected endpoints yet).
