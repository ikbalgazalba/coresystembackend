# Unit U-003 — Create multi-stage Dockerfile

## Goal

Create a multi-stage `Dockerfile` that builds the app with `./mvnw` inside an `eclipse-temurin:21-jdk` builder stage and runs only the JAR on an `eclipse-temurin:21-jre` runtime stage — no Maven, no source, no secrets in the runtime image.

## Context

- **Vault source:** `02-architecture.md` §Containerization layer; `04-flows.md` Flow 1; `05-decisions.md` ADR-AP-2; `06-constraints.md` HR-1, HR-2
- **Binding refs:** C-010 (Dockerfile NEW), OQ-AP-2 (container port), OQ-AP-3 (trust-store mount target)
- **Why this scope:** the Dockerfile is a self-contained build artifact; compose (U-004) depends on it but the Dockerfile itself has no cross-file coupling.
- **Starterkit context:** none (Dockerfile is infra, not Spring code).

## Anchors

- `binding.md C-010` — Dockerfile confirmed absent at repo root → NEW.
- `codebase-map.md §7` — `pom.xml` artifactId `coresystembackend`, version `0.0.1-SNAPSHOT` → JAR name `coresystembackend-0.0.1-SNAPSHOT.jar`; `spring-boot-maven-plugin` present → executable JAR. `mvnw` wrapper committed.
- `codebase-map.md §7 snapshot_repo` — builder stage REQUIRES network to `repo.spring.io/snapshot` (Boot 4 parent).

## Hard rules

- file Dockerfile MUST exist after bolt
- DO NOT add new pom.xml dependencies
- DO NOT modify src/main/java/com/coresystem/coresystembackend/config/SecurityConfig.java

> Hard-rule provenance: HR-1/HR-2 (constitution §G-001/§D-004/§D-005 — no secret in image, runtime `eclipse-temurin:21-jre`, builder uses `./mvnw`); SecurityConfig immutability from binding C-004; mvnw/JAR facts from codebase-map §7.

## Anti-patterns

- Do NOT bake `.env`, secrets, or the trust store into the image (HR-1, §D-002, §D-004) — `.dockerignore` (U-004) excludes them; secrets come only from `env_file` at run; trust store only via volume mount.
- Do NOT use a host-installed Maven (HR-2, §D-005) — use `./mvnw` in the builder stage.
- Do NOT use a single-stage image (ADR-AP-2) — runtime must be `eclipse-temurin:21-jre` (JRE only).
- Do NOT run tests in the image build (`-DskipTests`) — tests run as a separate gate; keeps the build fast + reproducible.
- The `eclipse-temurin:21-jre` image lacks `curl`/`wget` by default (advisor caveat) — if a curl-based healthcheck is desired, install it in the runtime stage OR (preferred) leave the healthcheck mechanism to compose (U-004) using a Java-based/`CMD-SHELL` probe. Do NOT assume curl exists.
- Do NOT pin an EXPOSE port as fact — port is OQ-AP-2 (default 8080). Use `EXPOSE 8080` with a comment that it is overridable via `SERVER_PORT`.

## Implementation steps

1. **Resolve OQ-AP-2** (container port): default `8080` (Spring Boot default; `application.yaml` sets no `server.port`). `EXPOSE 8080`. Document override via `SERVER_PORT` env. Record decision.
2. **Resolve OQ-AP-3** (trust-store mount target): choose a container path for the mounted trust store, e.g. `/opt/bankmega-truststore/bankmega-truststore.p12`. This is a human decision (blocking OQ) — pick a conventional path + document it; the compose unit (U-004) mounts to this path and sets `JAVA_OPTS=-Djavax.net.ssl.trustStore=/opt/...`. Record the chosen path.
3. Stage 1 `builder`: `FROM eclipse-temurin:21-jdk AS builder`; `WORKDIR /build`; `COPY pom.xml mvnw mvnw.cmd .` + `.mvn/` (if present); `COPY src/ src/`; `RUN chmod +x mvnw && ./mvnw -B clean package -DskipTests`.
4. Stage 2 `runtime`: `FROM eclipse-temurin:21-jre`; `WORKDIR /app`; `COPY --from=builder /build/target/coresystembackend-0.0.1-SNAPSHOT.jar app.jar`; `EXPOSE 8080`; `ENTRYPOINT ["sh","-c","java $JAVA_OPTS -jar app.jar"]` (so `JAVA_OPTS` env injects trust-store flags at run).
5. Verify build + that the runtime image contains no source/mvn.

## Acceptance criteria

- `docker build -t coresystembackend:u003 .` succeeds (builder runs `./mvnw -B clean package -DskipTests`).
- Runtime image (`eclipse-temurin:21-jre`) contains the JAR at `/app/app.jar` and NO `src/`, NO `.mvn/`, NO `mvnw`, NO `pom.xml`, NO `.env`.
- No secret string appears in any image layer (`docker history` + a secret-scan grep).
- `EXPOSE 8080` present (OQ-AP-2 default); `JAVA_OPTS` honored in ENTRYPOINT (OQ-AP-3 trust-store path injectable).
- OQ-AP-3 trust-store mount target path decided + documented (e.g. in a comment + carried to U-004).
- `# SDD-PROVENANCE: U-003 | vault: .mega-sdd/vaults/api-platform | multi-stage Dockerfile` comment in Dockerfile.
