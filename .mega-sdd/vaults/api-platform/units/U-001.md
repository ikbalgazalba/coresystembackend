# Unit U-001 — Add springdoc-openapi dependency + OpenApiConfig

## Goal

Add the `springdoc-openapi-starter-webmvc-ui` dependency (version per OQ-AP-1) and a minimal `OpenApiConfig` `@Configuration` class so the existing `POST /api/auth/dologin` endpoint + DTOs are auto-documented and served at `/v3/api-docs` + `/swagger-ui.html`.

## Context

- **Vault source:** `02-architecture.md` §API documentation layer; `04-flows.md` Flow 3; `05-decisions.md` ADR-AP-1
- **Binding refs:** C-007 (springdoc absent → to be added), C-008 (OpenApiConfig NEW), C-014 (doc paths NEW), OQ-AP-1 (version), OQ-AP-5 (prod gating)
- **Why this scope:** documentation layer is self-contained — one dep + one config class. No endpoint modification (springdoc introspects annotations). SecurityConfig permitAll for the doc paths is handled in U-002 (health/actuator unit also touches SecurityConfig) to avoid two units editing SecurityConfig.
- **Starterkit context:** `starterkit-context.yaml` patterns.controller (config location confirmed), libs (spring-boot-starter-web present).

## Anchors

- `codebase-map.md §7` — springdoc-openapi confirmed ABSENT (C-007); to be added.
- `codebase-map.md §3` — `POST /api/auth/dologin` (`AuthUserController.java:54-55`) is the endpoint to document (C-001).
- `codebase-map.md §2` — DTOs `LoginRequest`/`JwtResponse`/`MessageResponse` (C-002) auto-documented.

## Hard rules

- DO NOT modify src/main/java/com/coresystem/coresystembackend/controller/AuthUserController.java
- DO NOT modify src/main/java/com/coresystem/coresystembackend/dto/LoginRequest.java
- DO NOT modify src/main/java/com/coresystem/coresystembackend/dto/JwtResponse.java
- DO NOT modify src/main/java/com/coresystem/coresystembackend/dto/MessageResponse.java
- file src/main/java/com/coresystem/coresystembackend/config/OpenApiConfig.java MUST exist after bolt
- DO NOT add new pom.xml dependencies
- DO NOT modify src/main/java/com/coresystem/coresystembackend/config/SecurityConfig.java

> Hard-rule provenance: HR-7 (constitution §D-003 — springdoc is the only new dep; actuator belongs to U-002); controller/DTO immutability from binding C-001/C-002 (codebase-map §2/§3); `starterkit-context.yaml` §patterns.controller (config/ location).

## Anti-patterns

- Do NOT add an OpenAPI security scheme (Bearer JWT) in v1 — deferred per Q&A (00-index.md Out of Scope; ADR-AP-5).
- Do NOT invent a springdoc version — resolve OQ-AP-1 first (Boot 4.1.1-SNAPSHOT compatibility); if unresolved, halt and request PO/architect pin. *(anti-halu)*
- `LoginRequest.pass` will appear in the spec — acceptable for internal UAT; prod Swagger UI gating tracked by OQ-AP-5 (leave a clean seam: `springdoc.swagger-ui.enabled` property).

## Implementation steps

1. **Resolve OQ-AP-1** (springdoc version): look up the springdoc-openapi version compatible with Spring Boot 4.1.1-SNAPSHOT (Boot 4.x). If no compatible release/milestone is found, HALT and surface for PO/architect (do NOT guess a version). Record the chosen version + rationale in a `decisions.md`-style note in the commit.
2. Add `springdoc-openapi-starter-webmvc-ui` dependency to `pom.xml` `<dependencies>` (version from step 1). Keep existing deps untouched.
3. Create `src/main/java/com/coresystem/coresystembackend/config/OpenApiConfig.java` as a `@Configuration` class exposing an `OpenAPI` `@Bean` with: API title `coresystembackend API`, description, version `0.0.1-SNAPSHOT` (matches pom.xml `<version>`). Use constructor-style / `@Bean` per pack idiom (no field injection).
4. Leave a seam for prod gating (OQ-AP-5): do NOT hard-disable the UI; rely on the `springdoc.swagger-ui.enabled` property (set in U-005 prod profile). No code change needed beyond the config class — springdoc reads the property.
5. Run the context-loads test + manual spec fetch.

## Acceptance criteria

- `./mvnw -q test -Dtest=CoresystembackendApplicationTests#contextLoads` passes (context boots with springdoc on classpath).
- `GET /v3/api-docs` returns valid OpenAPI 3 JSON containing `paths."/api/auth/dologin"` and schemas `LoginRequest`, `JwtResponse`, `MessageResponse`.
- `GET /swagger-ui.html` loads the Swagger UI (note: full permitAll for these paths is wired in U-002; this unit's manual test runs against the dev profile where the paths may need temporary permit — OR run the spec test with the path permitted. Coordinate: U-002 MUST add `/v3/api-docs`, `/swagger-ui.html` to permitAll before the manual UI test passes without auth.)
- OQ-AP-1 resolved (version pinned with rationale) OR explicitly halted for PO input.
- No existing source file modified (controller, DTOs, SecurityConfig untouched).
- `// SDD-PROVENANCE: U-001 | vault: .mega-sdd/vaults/api-platform | springdoc-openapi + OpenApiConfig` trailing comment in OpenApiConfig.java.
