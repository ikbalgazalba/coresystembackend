# Unit U-002 — Add actuator health + additive SecurityConfig permit rules + externalize CORS

## Goal

Add `spring-boot-starter-actuator`, expose `/actuator/health`, and make ONLY additive permitAll changes to `SecurityConfig` for the health + doc paths (`/actuator/health/**`, `/v3/api-docs`, `/swagger-ui.html`). Additionally externalize the hardcoded CORS origins to a `${CORS_ALLOWED_ORIGINS}` env placeholder (HR-8) so container/prod browser clients are not rejected.

## Context

- **Vault source:** `02-architecture.md` §Health / observability layer + API contracts + CORS coupling note; `04-flows.md` Flow 4; `05-decisions.md` ADR-AP-4; `06-constraints.md` HR-3, HR-5, HR-8
- **Binding refs:** C-004 (SecurityConfig IMPLEMENTED, additive edit), C-007 (actuator absent → add), C-015 (health path NEW), OQ-AP-8 (actuator exposure surface), OQ-AP-9 (CORS externalization)
- **Why this scope:** SecurityConfig is the single file that needs the permitAll additions + CORS change; actuator is its natural pair (health probe needs the permit). Grouping avoids two units editing SecurityConfig.
- **Starterkit context:** `starterkit-context.yaml` auth (SecurityFilterChain bean, STATELESS), authz (url_rules /api/auth/** permitAll, anyRequest authenticated). CORS origins at `SecurityConfig.java:89`.

## Anchors

- `binding.md C-004` — `SecurityConfig.java:42` exists; permitAll is only `/api/auth/**` (`SecurityConfig.java:60`), `anyRequest().authenticated()` (line 61). EDIT is additive (HIGH confidence).
- `codebase-map.md §6` — existing `corsConfigurationSource()` hardcodes `http://localhost:3000`, `http://localhost:8080` (`SecurityConfig.java:89`).
- `binding.md C-007` — actuator ABSENT → add.

## Migration notes

- KEEP: `/api/auth/**` permitAll, `anyRequest().authenticated()`, `SessionCreationPolicy.STATELESS`, CSRF-disabled-with-comment, delegating `PasswordEncoder` bean, `RestClient.Builder` bean, CORS bean structure.
- ADD: `.requestMatchers("/actuator/health/**").permitAll()`, `.requestMatchers("/v3/api-docs", "/swagger-ui.html", "/swagger-ui/**").permitAll()` — BEFORE `anyRequest().authenticated()` (matcher order matters).
- ADD: actuator dependency to pom.xml.
- MODIFY: `corsConfigurationSource()` — replace `setAllowedOrigins(List.of("http://localhost:3000", "http://localhost:8080"))` with `setAllowedOrigins(...)` driven by `${CORS_ALLOWED_ORIGINS}` (split the env value on comma). Keep `setAllowedMethods`, `setAllowedHeaders`, `registerCorsConfiguration("/**", ...)` unchanged.

Actuator health exposure scope = OQ-AP-8 (default health-only, enforced via `management.endpoints.web.exposure.include=health` set in U-005 profiles).

## Anti-patterns

- Do NOT remove or weaken the existing `/api/auth/**` permitAll or `anyRequest().authenticated()` (HR-5) — only ADDITIVE matchers.
- Do NOT expose actuator endpoints beyond health unless OQ-AP-8 resolves to a wider surface (default: health only — `management.endpoints.web.exposure.include=health`).
- Do NOT replace the trust store with a trust-all manager (constitution §B-006) — not in scope here but do not regress.
- Do NOT hardcode CORS origins in any profile; the prod origin MUST come from `${CORS_ALLOWED_ORIGINS}` (HR-8). A dev default may live in application-dev.yaml (U-005).

## Implementation steps

1. Add `spring-boot-starter-actuator` to `pom.xml` `<dependencies>`.
2. In `SecurityConfig.filterChain`, insert (before `anyRequest().authenticated()`): `.requestMatchers("/actuator/health/**").permitAll()` and `.requestMatchers("/v3/api-docs", "/swagger-ui.html", "/swagger-ui/**").permitAll()`. Preserve all existing chain elements + comments.
3. In `corsConfigurationSource()`, replace the hardcoded origins list with origins parsed from `${CORS_ALLOWED_ORIGINS}` (env var, comma-separated). Use `@Value` or read via the existing config injection. Keep the rest of the CORS config. If `CORS_ALLOWED_ORIGINS` is unset, fail-fast is acceptable (consistent with §D-002 pattern) — but coordinate with U-005 which sets a dev default.
4. Resolve OQ-AP-8: default `management.endpoints.web.exposure.include=health` (health only). Set this in the base/ prod config (U-005 sets per-profile; this unit can set the baseline in `application.yaml` OR defer to U-005 — prefer U-005 to keep this unit focused on SecurityConfig). Decision: leave exposure to U-005 profiles; this unit only adds the permitAll matcher + actuator dep.
5. Update `SecurityConfigTest` if it asserts the exact matcher set (add assertions for the new permitAll paths). Run tests.
