# Unit U-004 — Create docker-compose.yml + .dockerignore

## Goal

Create `docker-compose.yml` (app service: build from U-003's Dockerfile, `env_file: .env`, trust-store volume mount, port publish, actuator healthcheck) and `.dockerignore` (exclude secrets/build-noise from the build context). Wire env + trust store at run time — never baked into the image.

## Anti-patterns

- Do NOT use `environment:` for secret literals (HR-1, G-003) — use `env_file: .env` (gitignored). Non-secret runtime knobs (SPRING_PROFILES_ACTIVE, JAVA_OPTS, CORS_ALLOWED_ORIGINS, TRUSTSTORE_PATH) MAY use `environment:` referencing `${...}` from the host env or `.env`.
- Do NOT bake the trust store into the image — mount it as a volume (G-003, HR-1).
- Do NOT add a `postgres` service unless OQ-AP-4 resolves to wanting one (default recommendation: env-only wiring to existing UAT DB at 10.95.1.43 — no postgres service). If OQ-AP-4 is ACCEPTED as env-only, OMIT the postgres service.
- Do NOT assume `curl` exists in the runtime image (advisor caveat) — use a healthcheck that works on the JRE image. Options: a `CMD-SHELL` Java probe, OR install wget/curl in U-003's runtime stage. Prefer: `test: ["CMD-SHELL", "java -version >/dev/null 2>&1 || exit 1"]` is NOT a real health check; instead use `wget -q -O- http://localhost:${SERVER_PORT:-8080}/actuator/health || exit 1` ONLY if wget is present, else a tiny Java probe. Coordinate with U-003's curl decision.
- Do NOT set resource limits as fact — OQ-AP-6 is `recommend` (fallback: no limits). If limits are added, cite OQ-AP-6 acceptance.

## Implementation steps

1. **Resolve OQ-AP-4** (postgres in compose): recommendation = env-only wiring to existing UAT (no postgres service). ACCEPT this recommendation unless the PO overrides. Document the decision.
2. Create `.dockerignore` excluding: `target/`, `.git/`, `.mega-sdd/`, `.env`, `*.log`, `.idea/`, `.vscode/`, `preview.webp`, `node_modules/`, `Dockerfile` (optional), `docker-compose.yml` (optional). KEEP `.env.example` (template, not secret) + `run-app.sh` (harmless) — or exclude run-app.sh (host-only). Decide: exclude run-app.sh (host launcher, irrelevant in container).
3. Create `docker-compose.yml`:
   - `services.app`: `build: .`; `image: coresystembackend:latest`; `env_file: .env`; `environment:` `SPRING_PROFILES_ACTIVE=prod`, `JAVA_OPTS=-Djavax.net.ssl.trustStore=/opt/bankmega-truststore/bankmega-truststore.p12 -Djavax.net.ssl.trustStorePassword=${TRUSTSTORE_PASSWORD:-changeit} -Djavax.net.ssl.trustStoreType=PKCS12`, `CORS_ALLOWED_ORIGINS=${CORS_ALLOWED_ORIGINS}`; `ports: "8080:8080"` (OQ-AP-2 default); `volumes: - ${TRUSTSTORE_HOST_PATH:-/home/ikbalgazalba/.ssl-truststores/bankmega-truststore.p12}:/opt/bankmega-truststore/bankmega-truststore.p12:ro`; `healthcheck:` per the curl/probe decision (interval/timeout/retries set).
   - NO `postgres` service (OQ-AP-4 env-only).
   - Optionally `deploy.resources.limits` (OQ-AP-6 — default omit; fallback "no limits").
4. Validate `docker compose config -q`.

## Acceptance criteria

- `docker compose config -q` passes (valid syntax).
- `.dockerignore` excludes `.env`, `target/`, `.git/`, `.mega-sdd/` (HR-1 — no secret/noise in build context).
- compose `app` service uses `env_file: .env` (no secret literals in `environment:`); trust store is a volume mount (not in image).
- `SPRING_PROFILES_ACTIVE=prod` set (coordinates with U-005 prod profile).
- `CORS_ALLOWED_ORIGINS` + `TRUSTSTORE_*` env wired (coordinates with U-002 CORS externalize + U-003 mount path).
- Healthcheck references `/actuator/health` and works on the JRE image (curl/probe decision resolved, no silent curl assumption).
- OQ-AP-4 resolved (postgres decision documented); OQ-AP-6 resolved (limits decision documented).
- `# SDD-PROVENANCE: U-004 | vault: .mega-sdd/vaults/api-platform | docker-compose + .dockerignore` comment in docker-compose.yml.
