# Unit U-005 — Create Spring profiles (dev + prod)

## Goal

Create `application-dev.yaml` (local: localhost CORS default, Swagger UI on) and `application-prod.yaml` (container: externalized CORS/trust-store, actuator health-only exposure, Swagger UI gated off per OQ-AP-5) — overriding ONLY profile-specific keys. Base `application.yaml` secret-placeholder fail-fast behavior is preserved (HR-4).

## Context

- **Vault source:** `02-architecture.md` §Health / observability layer (Profile config); `04-flows.md` Flow 5; `05-decisions.md` ADR-AP-4, ADR-AP-5; `06-constraints.md` HR-4
- **Binding refs:** C-006 (base application.yaml IMPLEMENTED — preserve), C-013 (profile YAMLs NEW), C-015 (health path), OQ-AP-5 (Swagger UI gating), OQ-AP-8 (actuator exposure), OQ-AP-9 (CORS default per profile)
- **Why this scope:** both profile YAMLs are small config files; grouped as one unit (≤5 files, small complexity). No code coupling beyond referencing keys consumed by U-002 (CORS) + U-004 (compose env).
- **Starterkit context:** none (config files).

## Anchors

- `binding.md C-006` — base `application.yaml` exists; secret `${VAR}` placeholders have NO defaults (fail-fast); `jwtExpirationMs: 86400000` is a literal default (advisor APB-001 — keep it, do NOT convert to placeholder).
- `binding.md C-013` — profile YAMLs confirmed absent → NEW.
- `binding.md C-015` + U-002 — `/actuator/health` exposed; OQ-AP-8 default = health only.

## Hard rules

- file src/main/resources/application-dev.yaml MUST exist after bolt
- file src/main/resources/application-prod.yaml MUST exist after bolt
- DO NOT modify src/main/resources/application.yaml
- DO NOT add new pom.xml dependencies

> Hard-rule provenance: HR-4 (constitution §G-006 — profile YAMLs override only profile-specific keys; base application.yaml fail-fast preserved); base-config immutability from binding C-006 (advisor APB-001 — `jwtExpirationMs` literal default kept).

## Anti-patterns

- Do NOT move secret values into profile YAMLs (HR-4, §D-002) — secrets stay in base `${VAR}` placeholders + `.env`.
- Do NOT override the base fail-fast behavior — profile YAMLs set ONLY profile-specific keys (CORS origins, Swagger UI toggle, actuator exposure, server port override if any).
- Do NOT set `jwtExpirationMs` in a profile (advisor APB-001 — it is a non-secret literal default in base; leave it).
- Do NOT enable full actuator exposure in prod (OQ-AP-8 default = health only).
- Do NOT leave Swagger UI enabled in prod without resolving OQ-AP-5 — recommendation ACCEPT: disable UI in prod (`springdoc.swagger-ui.enabled=false`), keep spec at `/v3/api-docs` (internal).

## Implementation steps

1. Create `application-dev.yaml`:
   - `coresystem.cors.allowed-origins: http://localhost:3000,http://localhost:8080` (dev default; consumed by U-002's `${CORS_ALLOWED_ORIGINS}` — OR set the env-style key the SecurityConfig reads. Coordinate the key name with U-002.).
   - `springdoc.swagger-ui.enabled: true` (dev: UI on).
   - `management.endpoints.web.exposure.include: health` (dev: health only by default).
2. Create `application-prod.yaml`:
   - CORS origins: do NOT hardcode — rely on `${CORS_ALLOWED_ORIGINS}` env (compose U-004 sets it). If a profile-level default is needed, leave it absent (fail-fast if unset) per §D-002 discipline.
   - `springdoc.swagger-ui.enabled: false` (OQ-AP-5 ACCEPT — UI off in prod; spec still at /v3/api-docs).
   - `management.endpoints.web.exposure.include: health` (OQ-AP-8 default = health only).
   - Optionally `server.port: ${SERVER_PORT:8080}` (OQ-AP-2 — make explicit the port is env-driven; default 8080).
3. Verify base `application.yaml` is untouched; the context-loads test passes with each profile.
4. Confirm the CORS property key matches what U-002's `corsConfigurationSource()` reads (e.g. `@Value("${coresystem.cors.allowed-origins:}")`). Align the key across U-002 + U-005.

## Acceptance criteria

- `./mvnw -q test -Dtest=CoresystembackendApplicationTests#contextLoads` passes (base + profiles load).
- `application-dev.yaml` + `application-prod.yaml` exist and override ONLY profile-specific keys (CORS, swagger-ui.enabled, actuator exposure, optional server.port).
- Base `application.yaml` unchanged (diff empty); its secret `${VAR}` placeholders + `jwtExpirationMs` literal default intact (HR-4, advisor APB-001).
- prod profile disables Swagger UI (`springdoc.swagger-ui.enabled=false`) — OQ-AP-5 ACCEPT.
- prod + dev both expose `management.endpoints.web.exposure.include: health` (OQ-AP-8 default).
- CORS property key consistent with U-002's SecurityConfig reader (OQ-AP-9).
- `# SDD-PROVENANCE: U-005 | vault: .mega-sdd/vaults/api-platform | Spring profiles dev/prod` comment in each profile YAML.
